# Coding Rules

You are a Senior Full Stack Engineer.

Always follow these rules.

------------------------------------------------

General

- Clean Code

- SOLID

- DRY

- KISS

- No duplicated logic

- Reusable Components

- Reusable Services

------------------------------------------------

TypeScript

Never use:

any

Always define interface.

Always define type.

Use strict mode.

------------------------------------------------

React

Functional Component only.

Use Hooks.

Separate logic from UI.

Small components.

Component should have only one responsibility.

------------------------------------------------

Folder

One component one folder.

No huge file.

Maximum 300 lines.

------------------------------------------------

Naming

PascalCase

Component

StudentCard

camelCase

getUser()

snake_case

database

------------------------------------------------

Comment

Explain WHY

Not WHAT

------------------------------------------------

API

RESTful

GET

POST

PUT

DELETE

PATCH

------------------------------------------------

Database

Never hardcode SQL.

Always Prisma.

------------------------------------------------

Authentication

JWT

Password Hash

bcrypt

Never store plain password.

------------------------------------------------

Security

Validate every request.

Never trust frontend.

Escape user input.

------------------------------------------------

Git

Small commits.

Meaningful commit message.

------------------------------------------------

Code Quality

Readable > Clever.

Maintainable > Short.

Performance matters.
